專題簡介 Empowering Sustainable Audits with AI and RAG Technology
💡 專題核心精神與研究背景

在目前永續金融與企業合規的趨勢下，資訊的揭露品質與真實性成了關鍵。傳統 ESG 查核極度依賴企業的質性宣告，導致審計程序耗時且極難核對。

本專題開發的 EcoTrust AI v2.0 平台，首創將財務獲利硬指標 (ROE) 與由 AI 模型提取分析的永續承諾信心得分 (Confidence Score) 進行雙引擎融合。透過「去碳排化」的文本實質性審計，評估其永續申報的資訊透明度與數據具體度，解決生成式 AI 用於嚴謹財務審計時易產生的胡言亂語（幻覺）痛點，建立具備高度可信度的審計軌跡。

🔬 去碳排化文本審計
不依賴企業自主申報的排放數據，而是分析文本的數字密度、指標多樣性與承諾品質，量化其揭露誠信度。
📊 雙引擎交叉象限
整合 MariaDB 財務季度數據與 ESG 信心分數，進行 2D 走勢氣泡分析，直觀揭示企業永續轉型的實質成效。
⚡ 頁碼感知 RAG 顧問
自研 Bigram 分詞與頁面命中計分，對接本地 Qwen2.5:7b，生成之論點均能追溯原始 PDF 頁碼。
🚀 專題發表四大技術亮點
1. Smart Chunking 智慧切片
優先選取報告書中含有第三方驗證與 ISO 等「硬指標」段落，並結合等距採樣，代表全文送入 FinBERT 推理。
2. Gen-2 承諾生命週期提取
使用正則表達式自動辨識並分類出具備「量化數據」與「明確時限」的高信度承諾，作為實質核對的核心點。
3. RSS 背景輿情驗證機制
針對提取的高信度承諾，在背景自動生成檢索詞爬取 Google News 進行情緒標記，提供市場對永續行為的實質回饋。
4. 雙向網頁 PDF.js 動態跳頁
用戶在 RAG 面板點擊引用標籤 📄 [p.X]，右側 PDF 視窗即秒級跳轉至原文該頁，實現所答即所見。

數據預處理管道 (Data Pre-processing)

預處理設計 Reducing Context Load & Enhancing Cross-Company Comparison efficiency
📑 1. 單公司：PDF 頁碼解構索引化

為了使 RAG 能夠精確引述頁碼，我們拋棄了傳統的向量數據分片做法，而在預處理段使用 Python 的 pdfplumber 模組將報告文本按真實頁碼存成輕量化的頁面 JSON： 

# pdf_page_indexer.py 核心預處理邏輯
with pdfplumber.open(pdf_path) as pdf:
    pages_list = []
    for idx, page in enumerate(pdf.pages):
        text = page.extract_text()
        pages_list.append({
            "page": idx + 1,
            "text": text if text else ""
        })
    json.dump({"source_pdf": pdf_name, "pages": pages_list}, out_file)
📊 2. 跨公司：數據庫 Facts 壓縮與對齊

跨公司對比的關鍵在於不讓 LLM 閱讀數萬頁 PDF，而是使用預處理將 MariaDB 中的結構化 Facts（包含歷年 ROE、碳排放信心得分、量化承諾率、新聞情緒指標）導出成一份幾千字的 Markdown 對照報告（如 cross_company_analysis_report.md）。

技術優勢： 由於將大量 PDF 資訊預先提煉為結構化 Facts 報告，LLM 可以在不到 8,000 Tokens 的 Context 開銷下，直接執行多維度邏輯推理，絕不超載，杜絕了胡言亂語的發生。

🔍 1. 自適應中文 Bigram 滑動分詞器

在後端 PHP 環境中，我們自研了 Bigram 滑動分詞器（N-gram），搭配 ESG 行業領域字典與停用詞過濾，能從使用者問題中精準定位特徵詞，相容性高於一般向量庫： 

// chat_api.php 雙字滑動分詞核心代碼
$len = mb_strlen($cleanSegment, 'UTF-8');
for ($i = 0; $i < $len - 1; $i++) {
    $bigram = mb_substr($cleanSegment, $i, 2, 'UTF-8');
    if (preg_match('/[\x{4e00}-\x{9fa5}a-z0-9]/u', $bigram)) {
        $keywords[] = $bigram; // 產生雙字滑動詞
    }
}

核心評分演算法與 LaTeX 公式庫

演算法設計 Mathematical Scoring Models for Corporate Transparency Audits
🧠 1. 意圖強度得分 (Intent Strength Score, $I$)

使用本地部署的 FinBERT 中文情感模型，針對 100 個智慧切片 Chunk 進行積極機率與消極機率推理。單句 Chunk 得分公式如下：

$$S_{\text{chunk}, i} = \frac{P(\text{Positive}_i) - P(\text{Negative}_i) + 1}{2}$$

將 100 個 Chunks 的得分求算術平均，即為企業的**意圖強度得分 ($I$)**：

$$I = \frac{1}{N} \sum_{i=1}^{N} S_{\text{chunk}, i} \quad (N \le 100)$$
⚖️ 2. 誠信可靠性指數 (Credibility Index, $C$)

依據數字密度與 ESG 關鍵指標豐富度兩個歸一化指標加權計算：

$$N_{\text{norm}} = \min\left(1.0, \frac{N_s}{0.20} \times 1.5\right)$$
$$K_{\text{norm}} = \min\left(1.0, \sqrt{\frac{K_c}{15}}\right)$$

加權得到**可靠性指數 ($C$)**：

$$C = (W_{\text{numeracy}} \cdot N_{\text{norm}}) + (W_{\text{kpi}} \cdot K_{\text{norm}})$$

*(系統預設權重：數字密度子權重 $W_{\text{numeracy}} = 0.45$，關鍵指標子權重 $W_{\text{kpi}} = 0.35$)*

🛡️ 3. 原始總分與 Sigmoid 對比度增強

結合意圖分、可靠性指數與外部輿情風險值 ($R_e$)，求得原始總分 $S_{\text{raw}}$：

$$S_{\text{raw}} = (I \cdot W_{\text{INTENT}}) + (C \cdot W_{\text{CREDIBILITY}}) + \left[\left(1 - \frac{R_e}{100}\right) \cdot W_{\text{RISK}}\right]$$

為拉開不同揭露品質企業在 2D 圖表上的差距，使用 Sigmoid 對比度拉伸 作為最終誠信信心得分 $Y$（中心點 0.5，斜率 10）：

$$Y = \frac{1}{1 + e^{-10 \cdot (S_{\text{raw}} - 0.5)}}$$
